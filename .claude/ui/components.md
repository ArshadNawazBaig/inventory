# Components (Inventory)

> **Status:** 🟡 Seed · **Owner:** Frontend Lead · **Related:** [COMPONENT_LIBRARY.md](../../docs/COMPONENT_LIBRARY.md) (canonical, Phase 4) · [component-rules](./component-rules.md)

## Purpose
The catalog of components in `@stockflow/ui`. Authoritative spec:
[`/docs/COMPONENT_LIBRARY.md`](../../docs/COMPONENT_LIBRARY.md) (Phase 4).

## Inventory
- **Primitives:** Button, Input, Textarea, Checkbox, Switch, Select, Autocomplete, Radio, Slider, Label.
- **Overlays:** Modal, Drawer, Dialog, Dropdown, Tooltip, Popover, Toast, Command Palette.
- **Data:** Table, Data Grid, Pagination, Card, Badge, Avatar, Stats Card, Charts.
- **Navigation:** Sidebar, Navbar, Breadcrumb, Tabs, Accordion.
- **Forms & input:** Date Picker, Search Bar, Filters, File Upload.
- **Feedback:** Loading Skeleton, Empty State, Error State.
- **Domain:** Permission Wrapper, Role Badge, Status Badge.

## Each component must be
Reusable · fully typed · accessible (Radix-based) · responsive · dark-mode ready · variant-driven ·
themeable · documented · Storybook-ready. See [component-rules](./component-rules.md).

## Per-component documentation template
Anatomy → Props → Variants → States → Accessibility notes → Examples → Do/Don't.
